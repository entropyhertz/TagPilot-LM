LM Studio local captioning

A single-file browser application designed primarily for Windows and compatible with macOS and Linux. The default workflow keeps TagPilot and LM Studio on the same Windows PC. No paid API provider is required.

Windows quick start — same PC

Load a vision-capable model in LM Studio, open PowerShell, and start the local API server:

lms server start --port 1234 --cors

Confirm it is running with lms server status. In the TagPilot-LM repository directory, serve this file locally:

py -m http.server 8765

If py is unavailable, use python -m http.server 8765 or python3 -m http.server 8765.

Python's HTTP server may listen on all interfaces by default. On an untrusted network, append --bind 127.0.0.1 to restrict it to this PC.

Open http://127.0.0.1:8765/tagpilot.html and use http://127.0.0.1:1234 as the LM Studio URL.

Optional advanced mode — another device over a private LAN

On the Windows LM Studio host, run:

lms server start --bind 0.0.0.0 --port 1234 --cors

Confirm the server with lms server status, find the Windows IPv4 address with ipconfig, then enter http://<WINDOWS-IP>:1234.

LAN security warning: --bind 0.0.0.0 exposes LM Studio to the local network and --cors permits browser pages to call it.

Use this only on a trusted private network. Do not expose port 1234 through the router. Stop the server when it is not needed on an untrusted network. Use authentication when available, and never commit tokens.

localhost, 127.0.0.1, and ::1 are valid when LM Studio runs on this same PC. 0.0.0.0 is never a destination URL.

Only needed if authentication was separately configured on LM Studio. Kept in sessionStorage only; never commit tokens.
